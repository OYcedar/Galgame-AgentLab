# 发布 QA 流程

发布前必须完成以下检查：

1. 在干净的当前版本本体上覆盖发布文件。
2. 首次启动游戏。
3. 进入标题菜单。
4. 从 Start 进入正式对白。
5. 打开 Load、Save、Log、Config、Extra 等常见页面。
6. 检查选项分支和 Ctrl/Skip。
7. 确认没有阻塞型弹窗、黑屏、后台运行、闪退或乱码。
8. 抽查译文、标点、换行和字体显示。
9. 抽查人称、敬称和角色称呼一致性。

发布目录只包含玩家运行补丁需要的文件。工作产物保存在项目工作目录：

- `_work`
- `save` / `savedata`
- 解包素材
- 调试截图
- 临时 exe / dll
- 原始 CG、语音、BGM、视频封包，除非它们确实被修改且允许分发

建议把 QA 截图和测试日志保存到对应游戏的工作目录，而不是发布目录。
